# Security Policy

Although this is primarily a documentation repository, its examples may influence agents that can run commands, modify systems, handle customer data, or control physical devices.

## Report privately

Use a private security advisory or contact the maintainer privately for:

- credentials, private data, customer identifiers, or recordings committed by mistake;
- an example skill that can cause destructive or unauthorized actions;
- unsafe production-operation instructions;
- cross-tenant, OpenClaw, IoT, voice, or data-handling risks that should not be publicly disclosed before mitigation.

Do not open a public issue containing active secrets, exploitable production details, or personal data.

## Scope

Security review covers the repository's instructions, templates, workflows, and published case material. It does not certify third-party tools or guarantee that following the guide makes a system secure.
