# 第一部分 方法论重构
> *以下内容把原始实践体会整理成一个统一模型：四个平面负责控制闭环，十个阶段覆盖项目全生命周期。*

## 1. 四个平面：AI 原生工程的基本结构
传统开发容易把“写代码”视为中心活动。AI 编程之后，真正的中心变成了控制、执行、证据和学习之间的闭环。

## 2. 十阶段生命周期：一个环节都不能少
| **阶段** | **名称**           | **核心问题**                                         |
|----------|--------------------|------------------------------------------------------|
| 0        | 问题与价值定义     | 确认为什么做、为谁做、成功是什么，以及什么不做       |
| 1        | 需求规格           | 把想法转成场景、规则、边界、异常与验收标准           |
| 2        | 非功能需求与风险   | 明确性能、安全、可靠性、隐私、成本、兼容性和可运维性 |
| 3        | 设计与架构         | 完成 UI/UX、API、模块、数据、算法、部署与演进设计    |
| 4        | 任务拆分与代码方案 | 把设计拆成小批次，先审方案再执行                     |
| 5        | 实现与局部验证     | 在明确允许范围内生成代码，并立即进行静态与单元验证   |
| 6        | 系统测试与对抗验证 | 用自动化、端到端、契约、属性和回归测试证明行为       |
| 7        | 可观测性与问题诊断 | 通过日志、指标、追踪和最小复现定位真实运行过程       |
| 8        | 性能、发布与运维   | 基准、压测、迁移、灰度、监控、回滚与事故处理         |
| 9        | 复盘与 Skill 沉淀  | 把问题转成规则、测试、模板、Skill 和后续改进         |

小项目可以把多个阶段合并到同一份文档中，但不能让这些问题从流程中消失。例如，可以把需求、非功能要求和架构写在一个简洁设计说明中，却不能因为“项目小”就不定义验收标准或回滚办法。
