# 文档定位与阅读说明
> *这不是一份工具评测，也不是“提示词技巧大全”。它把一组来自真实项目的经验，重构为可复用、可检查、可迭代的 AI 原生软件工程方法。*

本文基于作者近一年的 AI 编程实践：前期主要使用 Claude，后期全面转向 Codex，复杂算法与从 0 到 1 的研究通常先在 ChatGPT Pro 中深入讨论，再由 Codex 结合本地代码库审核、细化并实现。项目覆盖业务系统、Mobile App、纯技术项目、底层硬件相关项目以及机器学习项目。

根据维护者目前的跨项目统计估算，Codex 累计生成了超过 30 万行应用代码，维护者没有亲手编写这些代码，并在个人实践中观察到显著效率提升。代码量与“约十倍效率”的统计方法尚未经过独立审计，后续版本将补充口径；本文把它们标记为个人实践估算，不作为行业基准或普遍结论。它们说明的是一个结构性问题：当人的工作重心从逐行编码转向系统控制时，软件工程的分工方式会发生根本变化。

## 当前方法论版本的新增内容
本版把最新实践进一步系统化，并补入公开发布方案。核心变化不是增加更多工具名称，而是把“如何在代码规模暴增之后仍保持控制”写成一套完整机制。

| **新增主题**       | **本版强化的核心观点**                                                                          |
|--------------------|-------------------------------------------------------------------------------------------------|
| AI 原生 Review     | 人工不再逐行修改海量代码；AI 分层审查架构、需求覆盖、实现细节与证据，人负责风险判断和最终责任。 |
| 测试工具基础设施   | pytest、Playwright、Maestro、硬件在环、性能基准等成为大规模 AI 代码的主要信任机制。             |
| 文档即控制面       | 把思考过程完整文字化，形成需求、设计、算法、数据、计划、测试、运维的连续证据链。                |
| Skill 分层与治理   | 从底层原则到技术栈、项目和任务质量门，持续把成功经验与缺陷根因固化为可验证 Skill。              |
| AI 可读可观察性    | 结构化日志、指标、追踪和诊断地图让 AI 能从运行过程快速定位问题，而不是盲目重试。                |
| 人的价值与软件经济 | 审美、架构、算法、取舍和责任更加重要；更低开发成本将扩大个性化和长尾软件需求。                  |
| 案例与公开发布     | 用真实项目作为方法证据，并给出中英文 GitHub 仓库、LinkedIn 系列内容和贡献治理方案。             |
