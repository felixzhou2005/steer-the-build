# 第四部分 模板、提示与更多思路
> *以下内容可以直接复制到项目中，也可以用来触发你对过去一年实践的更多回忆。*

## 29. 实用模板一：需求规格卡
| **字段** | **需要回答的问题**                                 |
|----------|----------------------------------------------------|
| 问题     | 我们要解决什么问题？对谁？当前损失或摩擦是什么？   |
| 目标     | 可观测的成功结果是什么？                           |
| 范围     | 本次包含什么，不包含什么？                         |
| 场景     | 主要用户旅程、输入、状态、输出和异常是什么？       |
| 规则     | 业务不变量、权限、并发、时间和数据规则是什么？     |
| 非功能   | 性能、安全、可靠性、兼容、成本和运维要求是什么？   |
| 验收     | 哪些示例、反例、测试、指标或人工体验可以证明完成？ |
| 风险     | 最可能失败在哪里？哪些决定不可逆？                 |
| 变更     | 与上一版相比改了什么，影响哪些设计与测试？         |

## 30. 实用模板二：代码方案审查卡
| **字段**   | **内容**                                             |
|------------|------------------------------------------------------|
| 任务目标   | 一句话说明本次只解决的问题与非目标                   |
| 影响范围   | 读取、修改、新增和删除的文件/模块                    |
| 接口变化   | API、类型、数据、配置、协议和兼容性                  |
| 实现步骤   | 按顺序说明数据流、控制流和状态变化                   |
| 简单性     | 为什么这是满足需求的最简单方案；删除了哪些不必要设计 |
| 失败处理   | 规格内错误、超时、重试、幂等、降级和恢复             |
| 证据       | 单元、契约、集成、端到端、属性、基准和人工检查       |
| 风险与回滚 | 高风险路径、迁移、观测、回滚或补偿                   |
| 停止条件   | 达到什么证据后结束；失败多久后回到设计               |

## 31. 实用模板三：问题诊断记录
| **字段** | **内容**                                    |
|----------|---------------------------------------------|
| 现象     | 预期、实际、影响范围、首次发生时间          |
| 复现     | 输入、数据、环境、版本、步骤、频率          |
| 证据     | 日志、指标、追踪、截图、查询计划、硬件信号  |
| 假设     | 候选根因及每个假设需要的验证                |
| 隔离实验 | 怎样切断依赖、缩小输入或对比基线            |
| 根因     | 完整因果链，而不是发生问题的最后一行代码    |
| 修复     | 为什么修复根因；是否引入新风险              |
| 回归     | 新增测试、监控、阈值或故障注入              |
| 流程改进 | 需求、设计、Skill、文档或质量门需要怎样改变 |

## 32. 实用模板四：Skill 规范
| **字段**   | **内容**                                    |
|------------|---------------------------------------------|
| 名称与版本 | 清晰场景名；版本号；最后更新时间            |
| 目的       | 该 Skill 解决什么重复问题，期望改善什么指标 |
| 适用范围   | 语言、框架、模块、任务类型、前置条件        |
| 不适用范围 | 高风险、特殊性能、遗留兼容或其他例外        |
| 强制规则   | 必须做什么，优先级和不可违反项              |
| 禁止模式   | 过度设计、额外兜底、无理由依赖等反模式      |
| 工作流程   | 计划、实现、验证、汇报和停止条件            |
| 输出格式   | 方案、差异、测试结果、风险和已知问题        |
| 验证方法   | 命令、测试、基准、审查人和通过阈值          |
| 正反例     | 理想输出、常见错误和纠正方式                |
| 维护规则   | 回归任务、变更记录、过期条件和负责人        |

## 32.1 实用模板五：AI 原生 Review 报告
| **字段**     | **应填写的内容**                                                      |
|--------------|-----------------------------------------------------------------------|
| **审查对象** | 仓库 / 分支 / Commit / 需求版本 / 架构版本 / 配置与数据版本           |
| **审查范围** | 架构、功能、安全、数据、并发、SQL、性能、异常、文档、发布和回滚       |
| **需求追踪** | 每条需求对应实现位置、测试和证据；列出未覆盖、歧义和范围外变更        |
| **问题记录** | 严重度、文件/模块、触发场景、证据、影响、根因假设和最小修复           |
| **工具证据** | 构建、pytest、Playwright、Maestro、静态分析、基准、迁移演练和日志链接 |
| **风险接受** | 由谁接受、为什么、有效期、补偿控制和后续任务                          |
| **复审结论** | 修复结果、回归证据、残余风险和是否允许合并 / 发布                     |

## 32.2 实用模板六：AI 可读的可观察性规格
| **字段**             | **示例问题**                                                       |
|----------------------|--------------------------------------------------------------------|
| **关键用户旅程**     | 哪些端到端流程必须能从一个 trace_id 还原？                         |
| **关键状态与不变量** | 哪些状态变化、金额、权限、设备或订单同步必须记录前后值与原因？     |
| **日志字典**         | 事件名、字段、数据类型、脱敏规则、级别、采样和保留期是什么？       |
| **指标与 SLO**       | 延迟、成功率、错误率、队列、模型质量、设备在线率和成本阈值是什么？ |
| **诊断地图**         | 每类症状先查哪些指标，再查哪些 Trace、日志和代码路径？             |
| **告警与恢复**       | 告警触发、责任人、自动动作、人工升级和回滚是什么？                 |
| **AI 使用边界**      | AI 可以读取哪些 Telemetry，哪些数据必须脱敏或禁止进入上下文？      |

## 32.3 实用模板七：公开案例模板
| **章节**       | **内容要求**                                         |
|----------------|------------------------------------------------------|
| **背景与约束** | 业务目标、用户、技术环境、时间、风险和不能公开的信息 |
| **为什么困难** | 最关键的不确定性、跨层问题或非通用取舍               |
| **人的决定**   | 架构、算法、审美、指标、风险和停止条件               |
| **AI 的执行**  | 研究、方案、编码、测试、Review、日志分析、部署与运维 |
| **失败与反例** | 至少一个真实失败、错误路径或盲目重试，说明为什么发生 |
| **证据与指标** | 周期、人工投入、测试、性能、缺陷、恢复时间和业务结果 |
| **方法沉淀**   | 新增了什么文档、测试、日志、Review 规则或 Skill      |
| **隐私与边界** | 匿名化、授权、平台条款、安全控制和仍未解决的问题     |

## 32.4 实用模板八：Skill 评估记录
| **字段**         | **内容**                                           |
|------------------|----------------------------------------------------|
| **Skill 元数据** | 名称、层级、版本、所有者、模型/工具兼容和适用项目  |
| **触发与不触发** | 何时加载、前置输入、例外、禁止场景                 |
| **规则与理由**   | 必须做、禁止做、为什么，以及冲突优先级             |
| **正例 / 反例**  | 合格产物、不合格产物和最小差异                     |
| **验证集**       | 历史失败、正常任务、边界任务和不应受影响的任务     |
| **评估结果**     | 遵循率、误触发、漏触发、质量收益、时间成本和副作用 |
| **变更记录**     | 新增、修改、回退、废弃的原因与证据                 |
