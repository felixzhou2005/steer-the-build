# Roadmap

## v0.1 — Foundation

- Bilingual README and method documentation.
- Four-plane control model and ten-stage lifecycle.
- AI-native review, testing, observability, skills, human role, and AI DevOps.
- Complete Chinese long-form methodology and English chapter-based guide.
- Bilingual engineering templates and initial related-work research.
- Contribution, governance, licensing, and publication system.
- Optional MkDocs/GitHub Pages site.

Local case, skill, and social drafts are intentionally outside the public v0.1 until their evidence and content are reviewed.

## v0.2 — Evidence

- Publish three deeply documented cases: home valuation, iOS app, and either AI receptionist or IoT control.
- Add sanitized architecture diagrams, test reports, traces, benchmarks, and failure narratives.
- Replace all maintainer TODOs with verified facts or explicit non-disclosure notes.
- Add a case comparison table using the same evidence fields.

## v0.3 — Skill library

- Validate base and domain skills against historical project failures.
- Add trigger/non-trigger tests and version compatibility notes.
- Introduce project-level skill packs for mobile, ML, voice, IoT, and OpenClaw service deployment.
- Document skill conflicts, precedence, deprecation, and ownership.

## v0.4 — Measurement

- Define a common measurement schema for cycle time, rework, defect escape, rollback, diagnosis time, model cost, and maintenance effort.
- Publish before/after data where evidence is available.
- Add independent testing and reviewer-agent experiments.

## v1.0 — Stable field guide

- At least five complete, evidence-backed cases across different project types.
- Stable glossary and terminology.
- Full bilingual parity for all core chapters.
- Community-reviewed contribution quality gates.
- Versioned releases of the guide, templates, and skills.
