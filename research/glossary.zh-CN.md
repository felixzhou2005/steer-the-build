# 术语表

## AI 编码代理

能够检查仓库、修改多个文件、调用工具、执行命令并迭代实现的模型驱动工具。包括本地 Coding Agent 和 Agentic IDE 工作流，但不等于无限制自治。

## AI 原生 Review

AI 系统化分析架构、需求、行为、代码、安全、性能和运行证据的 Review 过程。人的注意力按照风险分配，而不是按照总行数平均分配。

## AI 可读可观察性

让人和 Agent 都能关联、检索系统行为证据的日志、Trace、指标、事件和诊断工件。

## 控制面

由人拥有、用于定义意图、约束、架构、风险、验收和权限的产物与规则。在本方法中，控制面主要是文字化并进入版本控制的。

## 证据面

用于判断系统是否满足声明需求的测试、日志、Trace、指标、基准、快照、Review 报告和运行观察。

## 第一个偏差点

实际行为与预期因果链最早发生不一致的可观察位置。找到它通常比反复修补最终症状更有效。

## 人在控制中

人继续拥有目标、重要取舍、风险阈值、发布权限和后果。AI 可以在明确批准的边界内获得较高执行自治。

## 独立验证

验证 Oracle 或证据不只来自实现 Agent 的假设。包括业务不变量、Golden Dataset、差分测试、真机流程和独立编写的验收例子。

## 项目 AI 宪法

对所有 Agent 任务生效的版本化项目规则，覆盖架构、编码、安全、测试、可观察性、运维和完成标准。

## Skill

可复用、可版本化的 Agent 指令包。有效 Skill 应包含适用范围、触发条件、规则、输出契约、验证方式、例外和示例，而不只是几段 Prompt。

## Skill 分层

规则的组合层次：基础原则、领域 Skill、项目约束和任务指令。更具体的层可以专门化，但不应无声违反更高层的高风险原则。

## 软件 Sense

在问题完全显现前识别抽象是否合适、失败模式、维护风险、产品取舍和运行后果的积累能力。AI 可以放大这种能力，但不会自动把它赋予初学者。
