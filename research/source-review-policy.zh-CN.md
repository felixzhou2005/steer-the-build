# 资料审查规则

## 目的

外部资料用于验证可能变化的事实、定义成熟术语、比较相关工作，并避免把常见思想包装成独有结论。项目自身的实践判断必须与外部可验证事实明确区分。

## 资料优先级

按结论类型优先使用最强资料：

1. 官方标准、规范或政府出版物；
2. 官方项目文档和代码仓库；
3. 原始研究论文或第一方工程报告；
4. 可信的独立工程分析；
5. 社区讨论只作为实践样本或争议示例。

## 规则

- 记录资料标题、规范链接、访问日期和它支持的具体结论。
- 技术行为和安全建议优先使用一手资料。
- 比较中的推断必须标为推断，不能伪装成来源自身结论。
- 不把供应商营销材料当成独立质量证明。
- 不复制长段落，采用概括和链接。
- 工具版本、安全模型、许可证和平台行为等可变事实必须重新确认。
- 可信资料有实质冲突时要呈现分歧。
- 绝不使用外部资料填补维护者私有项目中的未知事实。

## 审查节奏

每次发布带 Tag 的版本前：

- 检查全部外部链接；
- 复核标记为版本敏感的资料；
- 检查上次发布后新增的相关工作；
- 确认差异化表述仍然准确；
- 如果资料变化导致建议变化，在 `CHANGELOG.md` 中记录。

## 添加资料

Pull Request 应说明：

- 支持哪一条结论；
- 为什么资料足够权威；
- 相关页面或章节；
- 资料是否稳定、是否需要周期复核；
- 哪些中英文页面需要同步修改。
