## 1. 决策摘要

用自然语言说明架构、核心取舍，以及为什么它适合这个特定场景。

## 2. 背景与约束

- 产品和业务约束：
- 负载和性能预算：
- 安全、隐私、合规和租户约束：
- 现有系统和迁移约束：
- 团队、运维、成本和时间约束：
- 本文档之外已经固定的决定：

## 3. 系统上下文

说明用户、运维人员、设备、第三方系统、数据源和信任边界。添加架构图时，要用文字解释每个节点和箭头。

## 4. 组件模型

|---|---|---|---|---|---|
| | | | | | |

每项职责应有明确 Owner。指出重复权限或状态归属不清的地方。

## 5. 关键流程

每个高风险或高流量流程都应记录：

1. 入口条件；
2. 身份验证和授权；
3. 状态变化；
4. 同步和异步边界；
5. 外部调用；
6. 超时、重试、幂等和去重；
7. 可观察点；
8. 失败响应和恢复；
9. 用户可见结果。
