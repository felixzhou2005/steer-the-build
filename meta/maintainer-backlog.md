# Maintainer backlog

## Priority 0 — before public launch

- Replace repository and author placeholders.
- Decide the public author name and contact policy.
- Review the 300,000-line statement and describe how it was estimated or counted.
- Select three case studies for immediate deepening.
- Remove any project detail that cannot be safely disclosed.
- Run all validation commands and inspect the generated site.

## Priority 1 — first three deep cases

Recommended sequence:

1. **Home valuation:** problem framing, baseline, data semantics, model choice, evaluation, first-divergence diagnosis.
2. **iOS app:** Figma intent, faithful implementation, state behavior, Maestro flows, device evidence.
3. **AI receptionist or IoT control:** system boundaries, authorization, state synchronization, fallback, audit, and human takeover.

For each case, add one diagram, one failure narrative, one evidence table, and one skill derived from the work.

## Priority 2 — make the method falsifiable

- Publish one case where the method failed or was too costly.
- Add counterexamples to the strongest theses.
- Define minimum evidence for small, medium, and high-risk projects.
- Compare two ways of performing independent AI review.
- Measure rework, lead time, escaped defects, recovery time, and operating cost—not only generated code volume.
